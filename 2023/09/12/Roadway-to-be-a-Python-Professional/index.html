About this module

Installation

This section will cover the installation of necessary applications, including Python itself and PyCharm Community Edition (IDE).

❓ Why do we need IDE?
IDE stands for Integrated Development Environment, which really is just a fancy term of code editor.
IDLE, the out-of-the-box code editor comes along Python installation, is usable, but with extremely poor developing experience.

Installation of Python

Semantic Versioning

Before we start to install Python, we need to understand what does each digit in version number means.

We will use terms like “Major version”, “Minor version”, and “Patches” throughout the tutorial. Keep in mind that last digit stand for patches.

Fresh Installation

  1. Open the Official Python website. (Click on the picture to zoom in!)

  2. Hover on the download button, and download the corresponding OS version of Python.

  3. Open the python installer. If you are using a school computer, UNCHECK the option where it says “admin privileges”. CHECK the option “Add python.exe to PATH”

    ❗ Important

    You MUST check Add python.exe to PATH. Forgetting to do so will result in inability to find SDK, and eventually you will need to manually add Python SDK to PATH.

    If you somehow did not choose the option, manually adding Python to PATH will be covered later in this section.

  4. Click on “Install Now”, and wait for the progress to finish. Once you see the following image, your installation is finished.

Upgrade

Since we are demonstrating the installation of Python 3.11.5, say if you have a different patch (like 3.11.4) installed, your installer will look like this:

Click on “Upgrade Now” and wait for upgrade to finish.

Testing Installation

Now the installation is finished. We need to test if the installation has been performed correctly. Just like checking your work after finishing all the test questions.

  1. Open a new console by pressing ⊞ WinR, type cmd, and hit enter.

    We will use so what called “Command Prompt” a lot! Remember that cmd stands for command!

  2. In the console, type python --version and hit enter. If the output matches with the version you installed, then you are all set!

Troubleshooting

  • What if it says “‘python’ is not recognized as an internal or external command” OR it outputs a different version number from what I installed?

    When windows cannot find your program, most likely it is because you did not add the executable (executables are applications, .exe files) to PATH.

    Scroll down to the section “Manually adding Python to PATH”

  • Other issues? Tag me in school!

Manually adding Python to PATH

  1. Search up “path” in the windows search bar. Click on “Edit environmental variables for your account.”

    If you have admin privileges, you can select “Edit for system”.

  2. Click on PATH and click Edit

  3. Find your Python executable and copy the path.

    1. Usually your python should be installed under this directory C:\Users\<YOUR USER NAME>\AppData\Local\Programs\Python
    2. Open the folder with the corresponding major version and minor version. For example, if you installed Python 3.11, then open the folder Python311
    3. Copy the path, and you are done with step 3.
    4. If you did not get a valid path from the previous steps, search up “Python” in windows search, find the version you installed, and click “open file location”
    5. If you cannot find your python version by doing this, try to reinstall Python.
    6. Right-click on the shortcut, and click I

      For Windows 11 this shortcut does not work. I “stands for open file location”, right-click on the file, display more options, and “open file location” should be there.

    7. After clicking I, it should take you to a folder containing Python executable. Copy that path.

  4. Once you have the path, go back to PATH editor, click on New, and paste the path into the entry.

  5. Select OK all the way to close PATH editor.

Now, restart your command prompt (or open a new one) and test with python --version
